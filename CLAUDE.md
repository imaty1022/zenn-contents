## Overview

This is a [Zenn](https://zenn.dev) content repository for managing technical articles published on the Zenn platform (Japanese tech blogging site). Content is managed via `zenn-cli`.

## Commands

```bash
# Preview articles locally in browser
npx zenn preview

# Create a new article (generates a random hash filename)
npx zenn new:article

# Create a new book
npx zenn new:book

# List articles
npx zenn list:articles
```

## Article Frontmatter

Every article in `articles/` requires this frontmatter:

```yaml
---
title: "記事タイトル"
emoji: "😀"          # Single emoji displayed as article icon
type: "tech"         # "tech" for technical articles, "idea" for opinion pieces
topics: ["tag1"]     # Up to 5 topic tags
published: false     # Set to true to publish
published_at: "2025-01-01 09:00"  # Optional: schedule future publication
---
```

## Structure

- `articles/` — Markdown articles, each named with a random hash (e.g., `3cfd5f957ecef7.md`)
- `books/` — Book-format content (chapters as separate files)

## Content Guidelines

- Article filenames are generated by `npx zenn new:article` and should not be renamed manually (Zenn uses the hash as the article ID)
- Keep `published: false` until the article is ready to go live
- Topics/tags must match Zenn's registered topics for proper categorization
